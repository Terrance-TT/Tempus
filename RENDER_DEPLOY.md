# Deploying to Render

## Step 1: Connect your repo
1. Go to [dashboard.render.com](https://dashboard.render.com)
2. Click **New +** → **Blueprint**
3. Connect your GitHub repo: `Terrance-TT/Tempus`
4. Render will read `render.yaml` and create all services automatically

## Step 2: Set secret environment variables
After the blueprint creates the services, go to each service and set the secret values:

### Backend (`tempus-api`) — set these in Render dashboard:
| Variable | Where to find it |
|----------|-----------------|
| `CLERK_SECRET_KEY` | Clerk Dashboard → API Keys |
| `CLERK_PUBLISHABLE_KEY` | Clerk Dashboard → API Keys |
| `VITE_CLERK_PUBLISHABLE_KEY` | Same as CLERK_PUBLISHABLE_KEY |
| `SESSION_SECRET` | Already auto-generated by Render |
| `GOOGLE_CLIENT_ID` | Google Cloud Console (optional) |
| `GOOGLE_CLIENT_SECRET` | Google Cloud Console (optional) |
| `AI_INTEGRATIONS_OPENAI_API_KEY` | OpenAI Dashboard (required for AI) |
| `STRIPE_SECRET_KEY` | Stripe Dashboard (optional) |

### Frontend (`tempus-web`) — set these:
| Variable | Value |
|----------|-------|
| `VITE_CLERK_PUBLISHABLE_KEY` | Same Clerk key from above |

**Note:** `DATABASE_URL` is auto-set by Render from the PostgreSQL database.

## Step 3: Update Clerk allowed origins
1. Go to [Clerk Dashboard](https://dashboard.clerk.com)
2. Add these to **Allowed Origins**:
   - `https://tempus-web.onrender.com` (your frontend URL)
   - `https://tempus-api.onrender.com` (your backend URL)

## Step 4: Custom domain (optional)
If you want to use `tempusfokus.com`:
1. In Render, go to **tempus-web** → **Settings** → **Custom Domains**
2. Add `tempusfokus.com`
3. Follow Render's DNS instructions
4. Update `VITE_API_BASE_URL` to `https://api.tempusfokus.com` or keep the onrender.com URL

## Architecture
```
User → tempus-web.onrender.com (React frontend)
              ↓
       API calls to
              ↓
    tempus-api.onrender.com (Node.js backend)
              ↓
       tempus-db (PostgreSQL)
```

## Troubleshooting
| Problem | Fix |
|---------|-----|
| Build fails with pnpm error | Check that `packageManager` in `package.json` is `"pnpm@10.13.1"` |
| Database connection error | Check `DATABASE_URL` is auto-set from the database service |
| Blank schedule page | Check browser dev tools for API errors |
| Auth not working | Verify Clerk keys and allowed origins |
